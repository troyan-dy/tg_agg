# /check — тесты, линтеры и предложение опубликовать

Скилл для проекта **tg_agg** (uv + pytest + ruff + mypy). Запускается вручную
после изменений в коде. Цель — быстро убедиться, что всё зелёное, и предложить
закоммитить/запушить.

## Шаги

Выполняй по порядку. Все команды — через `uv run` из корня проекта.

### 1. Линтер (ruff)

```bash
uv run ruff check .
```

Если есть ошибки — попробуй автофикс безопасных правил и покажи, что починилось:

```bash
uv run ruff check . --fix
```

Оставшиеся (не-автофиксимые, например длинные строки `E501`) правь вручную,
сохраняя стиль окружающего кода. Не глуши правила через `# noqa` без причины.

### 2. Типы (mypy)

```bash
uv run mypy
```

Чинить реальные проблемы типов. Заглушки `# type: ignore[code]` — только с
узким кодом ошибки и коротким комментарием почему (как сделано в
`app/config.py` для pydantic-settings).

### 3. Тесты (pytest)

```bash
uv run pytest -q
```

Если тесты падают — разберись и почини код или тест. Не удаляй и не «скипай»
тесты, чтобы получить зелёный прогон. Если менялось поведение — обнови/допиши
тесты в `tests/`.

### 4. Итог

Кратко отчитайся таблицей вида:

| Проверка | Результат |
|----------|-----------|
| ruff     | ✅ / ❌    |
| mypy     | ✅ / ❌    |
| pytest   | ✅ (N passed) / ❌ |

Если хоть что-то красное — **не предлагай коммит**, сначала почини и перепрогони.

### 5. Предложение опубликовать в git

Только когда все три проверки зелёные:

```bash
git status --short
```

- Если рабочее дерево чистое — скажи об этом, публиковать нечего.
- Если есть изменения — покажи `git diff --stat`, предложи короткое сообщение
  коммита (по сути правок, на русском, в стиле истории репозитория) и **спроси
  подтверждение** перед коммитом и пушем. Не коммить и не пушь без явного «да».

При подтверждении:

```bash
git add -A
git commit -m "<сообщение>"   # завершай строкой Co-Authored-By (см. ниже)
git push
```

Сообщение коммита заканчивай:

```
Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

## Подводные камни проекта

- Пакетный менеджер только **uv** — не зови `pytest`/`ruff`/`mypy` напрямую.
- `.env` в `.gitignore` — следи, чтобы он не попал в `git add` (проверяй вывод
  `git diff --cached --name-only`).
- Тесты офлайн: БД — in-memory SQLite, внешние вызовы (DeepSeek, Telegram,
  feedparser) замоканы. Сеть/Postgres для прогона не нужны.
- Дефолтная ветка для пушей — `main`.
